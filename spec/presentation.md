# anneal-dev Specification — Presentation

The framework prescribes no presentation: it mandates no look, and
instances may differ freely (`instantiation-guide.md` §2). This
slot records the little anneal-dev steers; the framework defines
the slot but never dictates its content.

## anneal-dev's presentation

- **Persona / name:** "anneal-dev" — plain working name; no persona,
  no character framing. (The corpus-maintainer operator is the
  instance's own author; a persona would add noise to a tool used on
  its own methodology.)
- **Menu wording:** framework default — `(a)nother cycle` /
  `(n)ext phase` (`modules.md` §1.1). No instance override.
- **Visual flourish:** none. The closed-artifact form
  (`modules.md` §1.1) is presented as specified — code-fenced search
  commands, file paths, tracker citations — with no added icons or
  headers.

Kept (not deleted) only to record that the instance deliberately
takes the framework defaults and declares no persona — a recorded
non-decision, so a later reader does not re-open the question.
